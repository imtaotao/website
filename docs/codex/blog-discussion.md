# 博客模块讨论记录

这个文档用于记录当前博客页面需求的讨论结果，后续再逐步补充实现方案和未决议题。

## 当前目标

- 在现有网站中新增博客页面。
- 第一阶段先做最小可用版本。
- 先沉淀需求和方案，再进入实现。

## 已确认结论

### 1. 内容来源

- 博客内容放在本地目录中管理。
- 内容和代码一起放在仓库内，不依赖数据库。
- 站点以静态展示为主。

### 2. 内容组织

- 第一版以标签作为主分类方式。
- 一篇文章可以有多个标签。
- 标签有独立页面，可通过标签页索引到对应文章列表。
- 目录结构不承担主分类职责，内容聚合主要依赖标签。

### 3. 渲染方案

- 使用 `MDX` 作为文章内容格式。
- 需要支持：
  - 数学公式
  - 代码高亮
  - 图片

### 4. 视觉方向

- 博客页面的 UI 风格要和当前简历页面完全分开。
- 博客视觉需要更有个性。
- 视觉方案单独讨论，不在当前最小需求范围内一次定完。

### 5. 时间字段

- 文章需要 `publishedAt`。
- 文章需要 `updatedAt`。
- 由于当前没有数据库，这些信息以静态方式维护。

### 6. 非目标

- 第一版不做评论系统。
- 第一版不做订阅系统。
- 第一版优先最简化实现。

### 7. 工程约定

- 通用逻辑可以放到`packages/kernel-blog/` 包承载博客相关通用能力。
- 博客文章内容放到 `app/src/content/blog/` 内维护。
- `kernel-blog` 负责博客通用能力，不直接承载文章内容目录。

### 8. 内容目录结构

- `app/src/content/blog/` 采用扁平的文章单元目录结构。
- 每篇文章在自己的独立目录中维护 `index.mdx` 与相关资源。
- 文章目录先不通过年份、系列等子目录表达分类语义。

### 9. Frontmatter 最小字段

- 第一版至少包含：`title`、`slug`、`tags`、`publishedAt`、`updatedAt`、`summary`、`cover`。
- `slug` 使用手写方式维护，不从文件名自动生成。
- `cover` 作为文章头图字段，首版可选，但整体设计优先按“有 cover”组织。

### 10. 标签字段规范

- `tags` 存储机器友好的稳定值，而不是直接存展示文案。
- 标签展示名称通过单独映射维护，不直接依赖文章内原始标签值。

### 11. 标签展示配置

- 标签展示信息采用集中配置方式维护。
- 配置放在 `packages/kernel-blog/` 内。
- 数据结构以对象映射为主。
- 标签配置首版保留：`label`、`description`、`order`。

### 12. 博客路由结构

- 第一版采用以下基础路由结构：
  - `/blog`
  - `/blog/:slug`
  - `/blog/tags`
  - `/blog/tags/:tag`
- 标签页使用独立路由，而不是仅作为列表页内的查询过滤条件。
- 首版不引入基于年份或分类层级的额外路由结构。

## 当前理解下的第一版范围

基于已确认信息，第一版更适合控制在以下范围：

- 博客列表页
- 博客详情页
- 标签索引页
- 本地 MDX 内容加载
- 基础文章元数据展示
- 基于标签的文章聚合
- 文章头图展示
- 数学公式、代码高亮、图片渲染

以下内容暂不纳入第一版必做范围：

- 评论
- 订阅
- 数据库
- 复杂互动能力

## 待后续继续讨论的议题

这些议题已经识别出来，但本轮不强行定：

- 列表页信息密度
- `cover` 图片资源命名和引用约定
- 详情页目录、上一篇/下一篇等能力
- SEO、RSS、sitemap 等静态站配套能力
- 博客整体视觉语言
- `kernel-blog` 包的职责边界
