# 博客内容能力 Demo

这篇文章专门用来放博客正文支持的各种 case。图片统一放在文章自己的 `image/` 文件夹里，正文直接写相对路径就行。

## 1. 基础正文

普通段落、`inline code`、[站外链接](https://example.com) 都应该自然工作。

> 如果内容系统写起来不顺手，页面再漂亮也很难持续更新。

- 一段简短说明
- 一组并列要点
- 一些穿插在正文里的链接和代码片段

---

## 2. 标题层级

二级标题和三级标题都会进入阅读结构里，所以正文里正常写就行。

### 2.1 小标题示例

正文不需要刻意“写得像 demo”，只要让未来的你看一眼就知道这个能力怎么用就够了。

## 3. 列表和代码块

```ts
type BlogArticleFrontmatter = {
  title: string;
  slug: string;
  tags: Array<string>;
  publishedAt: string;
  updatedAt: string;
  summary?: string;
  cover?: string;
};

const createBlogTagNavigation = (tag?: string) => {
  if (!tag) return { pathname: '/blog' };
  return { pathname: '/blog', search: `?tag=${tag}` };
};
```

## 8. 后续约定

- 单图优先用 Markdown 原生语法
- 多图排版用 `ImageGallery`
- `columns={2}` 适合四宫格
- `columns={3}` 适合六宫格或九宫格
- `summary` 现在是可选的，不写也能发文
