# 简历模块 UI（讨论记录）

本文件用于记录“简历页面 UI/交互/视觉风格”的讨论结论，方便后续实现桌面端/移动端两套渲染，以及导出 `PDF/JPG`。

## 目标

- 高颜值、有个性但不花哨：信息密度适中，重点突出。
- 两套渲染：不做响应式；桌面端与移动端分别设计与实现。
- 可导出：页面提供导出按钮，一键导出 `PDF` 或 `JPG`。

## 信息架构（页面区块）

- 基本信息：姓名/Title/所在地（可选）/邮箱（可选）/社交链接（GitHub、知乎等）。
- 简介：1-3 条要点。
- 技能：按分类展示；每项有熟练度（1-100）。
- 工作经历：按时间倒序；每段经历以要点列出“做过的事情/产出”。

## 视觉风格（已定）

### 总体审美：东方留白（Clean + 空间感 + 克制）

- 关键词：留白、轻、静、克制、层级、呼吸感。
- 设计目标：看起来“干净且独特”，但仍然是可读、可导出、可维护的工程实现。

### 方向

- 墨白（Ink & White）
  - 基调：纯白/偏暖白底 + 近黑墨色文字（避免纯黑刺眼）。
  - 点缀：仅灰度体系（不使用彩色强调），通过字号层级、字重、间距与对齐制造“现代感”。
  - 结构：用大留白 + 极淡分割线 + 精准对齐建立层级；不使用古典符号化元素（印章/题签/书法装饰）。
  - 现代感来源：更大胆的字号层级、紧凑而克制的字距、统一圆角与细描边规范（仅在必要处使用）。

### 关键设计约束（保证“留白美学”不变成“空”）

- 栅格：桌面端固定宽度内容列（例如 880-980px 范围），移动端固定单列；两套渲染都使用一致的“垂直节奏”。
- 留白：章节间距显著大于段落间距（例如 48/24 的层级差），用空间表达层级。
- 分割：优先用间距与对齐表达分组，分割线/边框尽量少且极淡。
- 色彩：仅灰度体系（避免色差与导出偏色）。
- 字体：中文优先用现代无衬线；标题可用更“书卷气”的字重/字间距，而不是强行书法字体。

> 注：时间轴不作为全局风格；如需要，可作为“经历区块”的局部结构（例如左侧细线 + 节点），且必须克制。

## 桌面端渲染（Desktop）

- 布局：左侧信息栏（基本信息/链接/技能）+ 右侧主内容（简介/经历）。
- 技能：用“科技感数字”展示熟练度（例如 `86/100`），不使用进度条/圆环作为主视觉。
- 经历：公司+岗位+起止时间置顶，下面 `highlights` 列表。

## 移动端渲染（Mobile）

- 布局：单列，先基本信息与链接，再简介、技能、经历。
- 技能：优先用“科技感数字”（例如 `86/100`），紧凑不占高度。
- 导出按钮：放在页面顶部（可 sticky）或菜单中，避免遮挡内容。

## 导出（PDF/JPG）

- 交互：提供“导出 PDF / 导出 JPG”按钮。
- 范围：导出内容与页面展示一致（不包含交互控件）。
- 版式：导出时锁定字体、字号、行距，避免跨平台差异导致换行抖动。
- 备注：JPG 建议支持 `1x/2x` 清晰度；PDF 建议支持 A4（一页版，不够一页时可分页）。

## 组件库选型建议（高颜值/可个性化）

以下均适用于 React 生态；更偏“高颜值 + 个性化”的组合一般是“Headless/原子组件 + 自定义主题”。

### 选型结论

- 本项目 UI 组件库选用：`Radix UI`。
- 原则：优先用 Radix 提供的无障碍与交互能力；当某些效果/交互无法满足时，用自研组件补齐（保持 `墨白` 设计语言与导出友好）。

- 备注：不引入其他“带强风格”的组件库，避免风格漂移；必要时直接自研实现所需效果。

补充：本项目不使用 `Aceternity UI`（避免强绑定特定动效风格与导出/维护风险）。

## 决策与待办

- 视觉风格：主风格确定为 `墨白`；强调“现代感”，不使用古典装饰元素。
- 留白尺度：确定为 `48/24/12`（章节/段落/列表）。
- 强调色：仅灰度（不使用彩色强调）。
- 技能熟练度：用“科技感数字”呈现（例如 `86/100`），不使用进度条/圆环作为主视觉。
- 字体与排版：暂不讨论（后续需要时再补）。
- 导出实现：暂不讨论（后续实现阶段再定技术选型与导出范围）。
