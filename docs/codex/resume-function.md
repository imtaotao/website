# 简历功能说明

这个文档用于记录当前简历模块的产品约定。

## 范围

- 根路由 `/` 为简历页面。
- 简历内容当前只做中文。
- 页面重点展示工作经历，以及每段经历中的具体产出。
- 页面需要支持一键导出 `PDF` 和 `JPG`。
- 不做登录，不做后台，不做在线编辑。

## 渲染策略

- 不依赖单一响应式布局覆盖所有场景。
- 桌面端和移动端在必要时分别维护独立渲染。
- 根据设备尺寸选择对应渲染方式。
- 导出结果应尽量与页面展示保持一致。

## 建议数据结构

- `basics`
- `summary`
- `skills`
- `openSourceProjects`
- `experiences`

推荐日期格式：`YYYY-MM`

## 建议数据来源

- 使用 YAML 作为事实来源。
- 默认路径：`app/src/content/resume.yaml`
- 如果工作经历内容过长，可以拆成多个 YAML 文件，再通过包含列表进行组合。

## 工作经历结构

每条经历建议包含：

- `company`
- `department?`
- `role`
- `startAt`
- `endAt`
- `businessHighlights?`
- `techHighlights?`
- `highlights`

经历顺序保持从新到旧。

## 交付约束

- 页面内容应保持语义化，并且可复制。
- 桌面端和移动端都要保证独立可读性。
- 打印和导出样式要尽量稳定，避免 PDF 排版漂移。
