最近几个月在工作里有一些比较集中的想法，主要都围绕前端部署、同构方案和性能观测。

## 关于 SSR 和 CSR 的冗余设计

最近因为在公司里做一个前端部署服务，用到了类似 Next 的框架。但是这个服务本质上是一个 SSR 服务，而且公司的基建确实经常出现问题，FaaS 的稳定性也不是很好，所以一直在想是不是要补一层 CSR 来预防一些问题。这个思路前面其实也提过，本质上还是在做冗余设计。

并且这块核心逻辑本身是比较内聚的，对于本地前端工程构建来说也是一个很好的结合点，可以尝试做出一些非常惊艳的 DX 体验。所以回头再看，这种纯 Node.js 的框架其实很难真正满足同构的需求。

## 关于 Hono

后来发现了 Hono。它使用 Web 标准，支持很多不同的 JavaScript 运行时环境，相比 Koa 之类的框架，确实会感觉更先进一些。

如果使用 Hono 来做同构，上层很多能力其实都可以抽象成 HTTP server 的模式，跑在多端，底层又已经接近纯 SDK 的形态，这件事本身就很舒服。

## 关于 server-timing

借助 `server-timing` 做精细化的数据统计，其实也很适合去区分不同国家和地区的站点性能，尤其是 TTFB 这个阶段。不同阶段的耗时在不同地区带来的影响差别其实很大，特别是国际化业务再叠加大陆用户时，这种感受会更明显。

这样一来，就可以更有针对性地在对应地区增加 CDN 节点和动态加速能力。不过大陆这边严格的审查和备案机制，也确实会让这件事变难很多。
