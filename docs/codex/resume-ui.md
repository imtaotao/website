# 简历界面说明

这个文档用于记录当前简历页面的视觉和交互方向。

## 界面目标

- 有质感、有辨识度，但不过度花哨。
- 信息密度适中，重点明确。
- 桌面端和移动端可分别实现。
- 版式要适合导出 `PDF` 和 `JPG`。

## 信息结构

- 基本信息
- 简介
- 技能
- 开源项目
- 工作经历

## 视觉方向

主方向：`墨白`

- 大留白
- 轻、静、克制
- 只使用灰度色系
- 通过间距、字重、字号、对齐建立层级
- 避免古典装饰性元素

## 布局建议

桌面端：

- 左侧放基本信息、链接、技能
- 右侧放简介和工作经历

移动端：

- 单列布局
- 先基本信息和链接
- 再简介、技能、工作经历

## 间距规则

- 章节间距要明显大于段落间距。
- 优先用留白表达层级，再考虑分割线或边框。
- 分隔元素保持稀疏且足够轻。

## 技能展示

- 优先用数字形式展示，比如 `86/100`
- 不把进度条或圆环作为主要视觉语言

## 导出说明

- 提供明确的 `导出 PDF` 和 `导出 JPG` 操作
- 导出内容不包含交互控件
- 导出时锁定字体、间距和行高
- JPG 支持 `1x` 和 `2x`
- PDF 以 A4 为目标，必要时允许分页

## 组件方向

- 需要基础 UI 组件时，优先考虑 `Radix UI`
- 避免引入视觉风格过强的组件库
- 在稳定基础组件之上自定义页面表现
