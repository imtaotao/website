# 做个人博客时，我会先删掉哪些布局动作

很多页面看起来“乱”，不是因为元素太多，而是每个元素都想强调自己。

个人博客尤其容易出现这个问题。标题想做大，摘要想写满，标签想做成按钮，封面图又想撑出一个强烈的开场。最后读者还没开始读，就已经先被一堆视觉信号打断了。

## 我通常先删三种动作

- 删掉过度突出的精选卡片
- 删掉悬浮感很重的阴影和边框
- 删掉过大的标题级差

这些东西单独看可能都没有问题，但堆在一起时，页面会变得很吵。

## 真正重要的是节奏

我更在意列表里的节奏是不是稳定：

- 日期是不是安静地待在上面
- 标题是不是能在两秒内扫过去
- 摘要是不是只补充必要信息
- 分隔是不是主要依赖留白而不是盒子

如果这几个点是稳定的，页面通常就已经舒服了一大半。

> 极简不是把东西做少，而是让每一层都知道自己该退到哪里。

## 适合个人博客的第一版做法

第一版最稳的方式通常是：

- 单列列表
- 固定阅读宽度
- 小号元信息
- 中等字号标题
- 弱化标签

这套做法没有很强的“设计表演”，但它会让内容先成立。
