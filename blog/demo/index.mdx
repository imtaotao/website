这篇文章专门用来放博客正文支持的各种 case。图片统一放在文章自己的 `image/` 文件夹里，正文直接写相对路径就行。

这是一段专门用于排版测试的中文说明。正文里如果同时出现中文、英文和代码片段，整体节奏和行高都应该保持自然，不要一眼看出是不同系统拼起来的。

## 1. 基础正文

普通段落、`inline code`、[站外链接](https://example.com) 都应该自然工作。**加粗**字体应该正常显示。

### 1.1 颜色示例

推荐用更短的写法 <Color c="green">`这是一段绿色文字`</Color>，正文里直接用预设颜色名就行。

目前内置了 8 个预设颜色：

- <Color c="gray">gray</Color>
- <Color c="blue">blue</Color>

- <Color c="cyan">cyan</Color>

如果偶尔需要临时颜色值，依然可以继续写 <Color color="#8b38aa">自定义颜色</Color>，但更推荐优先用预设色。

- 一段简短说明
- 一组并列要点
- 一些穿插在正文里的链接和代码片段

## 5. 数学表达式

行内公式可以直接写在正文里，比如欧拉恒等式 $e^{i\pi} + 1 = 0$，或者二次方程求根公式 $x = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}$。

块级公式适合推导和单独展示：

$$
\int_{-\infty}^{\infty} e^{-x^2} \, dx = \sqrt{\pi}
$$

多行公式也可以正常排版：

$$
\begin{aligned}
f(x) &= ax^2 + bx + c \\
f'(x) &= 2ax + b
\end{aligned}
$$

## 14. Callout

`Callout` 适合承载说明、提示、注意事项和完成状态。遇到普通 `blockquote` 不够明确的场景，可以优先用它。

<Callout tone="note" title="补充说明">
  适合放一些需要紧贴当前段落的背景信息、边界条件或上下文说明，不会打断正文节奏，但读者也不容易漏掉。
</Callout>

<Callout tone="tip" title="写作建议">
  这里面依然可以直接写 MDX，所以像 <code>FileTree</code>、<Color c="green">强调文字</Color> 或 <UrlLink href="https://imtaotao.me" label="站点链接" /> 这种内容都能直接放进来。
</Callout>

<Callout tone="warning" title="明确写清代价">
  适合放不可逆操作、迁移注意点，或者读者很容易想当然写错的地方。比把这类信息埋在长段落里更稳。
</Callout>

<Callout tone="success" title="适用场景">
  适合放完成结果、确认结论，或者在分步骤内容之后做一个短收束，让读者快速知道这一段最后得到了什么。
</Callout>

## 15. Steps

`Steps` 更适合教程、排障流程和操作说明。和普通有序列表相比，它更稳定，也更容易做统一样式。

<Steps title="发布一篇新的 demo 文章">
  <Step title="先准备文章目录">
    文章放在博客目录下，图片等本地资源尽量和文章放在一起，这样相对路径最省心，也不容易写错。
  </Step>
  <Step title="普通 markdown 不够时再上 MDX 组件">
    像 <code>Callout</code>、<code>FancyList</code> 或行内链接这种能力，优先用在能明显提升结构清晰度的地方，不要为了 demo 而 demo。
  </Step>
  <Step title="稳定后再做一次真实验证">
    等正文结构差不多定下来，再跑一次完整构建或预览。很多导入路径和 MDX 写法问题，靠肉眼看没有实际渲染来得快。
  </Step>
</Steps>

## 17. 后续约定

- 单图优先用 Markdown 原生语法
- 多图排版用 `ImageGallery`
- 外站音频入口用 `AudioEmbed`
- 外站视频入口用 `VideoEmbed`
- 段落内的音频小入口用 `AudioLink`
- 段落内的视频小入口用 `VideoLink`
- 网页 iframe 卡片用 `WebEmbed`
- 普通外链胶囊用 `UrlLink`
- GitHub 账号提及用 `GitHubMention`
- GitHub 仓库入口用 `GitHubRepo`
- 数学表达式用 `$...$` 写行内公式，用 `$$...$$` 写块级公式
- 对话记录用 `ChatThread`
- 摘要卡片用 `SummaryCards`
- 古诗词卡片用 `Poem`
- `columns={2}` 适合四宫格
- `columns={3}` 适合六宫格或九宫格
- `summary` 现在是可选的，不写也能发文
