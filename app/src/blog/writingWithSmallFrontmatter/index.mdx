# 我为什么喜欢给文章准备更小的 frontmatter

做静态博客时，很容易一开始就想把 frontmatter 设计得很完整。

比如专题、系列、状态、阅读时长、置顶、作者、来源、封面说明、SEO 补充字段……这些能力以后也许都会用到，但第一版通常不需要一次拥有它们。

## 先保留最稳定的字段

我更倾向于把第一版压缩到这些字段：

- 标题
- slug
- 标签
- 发布时间
- 更新时间
- 摘要
- 封面

这些字段基本已经能支撑列表页、详情页和标签页。

```ts
type ArticleFrontmatter = {
  title: string;
  slug: string;
  tags: Array<string>;
  publishedAt: string;
  updatedAt: string;
  summary: string;
  cover?: string;
};
```

## 字段越多，不代表结构越成熟

很多时候，字段变多只是把“不确定的需求”提前写进了内容层。

这会带来两个问题：

- 写文章时要填更多东西
- 后面删字段时会影响已有内容

我更喜欢先让字段数量小一点，等页面和使用习惯真的稳定下来，再慢慢往上加。

## 这样做的好处

- 内容录入成本更低
- 列表结构更容易保持一致
- 后续扩展时更知道自己为什么要加字段

对于个人博客来说，这种节奏通常比“先设计完整模型”更健康。
