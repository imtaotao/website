# 站点体验优化记录

这个文档记录当前站点在首页、博客、简历和静态站配套能力上的后续优化方向。它不是一次性重构清单，而是后续小步迭代时的优先级参考。

## 当前状态

- 站点默认使用黑色主题。
- 首页保持文字型个人介绍，入口主要嵌在正文链接中。
- 博客已经具备文章列表、标签过滤、搜索建议、主题切换、文章头图、代码高亮、代码复制、图片灯箱和回到顶部。
- 博客文章使用本地 `MDX` 与 frontmatter 维护，`pnpm codex:blog-check` 可校验 frontmatter、slug 和本地图片引用。
- 简历页面已经具备桌面端分页预览和 `PDF` 导出能力。
- 简历文档仍要求支持 `PDF` 和 `JPG`，当前 UI 只实现 `PDF`。

## 优先级

### P0：先补站点完整度

这些改动成本低，但会明显提升站点作为公开个人网站的完整度。

1. 补全 SEO 与分享信息。

   - 将 `app/index.html` 的 `lang` 从 `en` 调整为 `zh-CN`。
   - 首页、博客首页、文章页和简历页应设置各自的 `document.title`。
   - 增加基础 `description`、Open Graph 和 Twitter Card meta。
   - 文章页优先使用 frontmatter 的 `title`、`summary` 和 `cover` 生成分享信息。
   - 后续可补 `sitemap.xml`、`robots.txt` 和 RSS。

2. 提供非空白 404 页面。

   - 当前未知路由不应渲染空白。
   - 404 页面保持简单：说明页面不存在，并提供返回首页和博客入口。

3. 增加全站主题切换入口。
   - 主题状态已经由 `kernel-shared` 统一管理，后续主题按钮应抽到共享组件。
   - 首页和简历页也应提供轻量主题切换入口，避免只有博客页面能切换主题。
   - 按钮视觉保持克制，不抢正文注意力。

### P1：提升博客阅读体验

这些改动直接影响文章阅读和站内流转。

1. 文章页增加目录与阅读进度。

   - Markdown 标题已经生成稳定 `id`，可在文章页消费标题列表。
   - 桌面端可以放右侧目录，移动端可做折叠目录或不展示。
   - 目录应支持当前章节高亮。
   - 顶部可加细阅读进度条，但不要做过重的视觉装饰。

2. 文章页增加上一篇 / 下一篇。

   - 按 `publishedAt` 倒序中的相邻文章计算。
   - 文章底部展示即可，不建议放进浮动按钮组。
   - 目标是让读完文章后有自然的继续阅读路径。

3. 文章 meta 展示更新时间。

   - `updatedAt` 已经存在于 frontmatter。
   - 当 `updatedAt` 不同于 `publishedAt` 时显示“更新于 yyyy.mm.dd”。

4. 标题锚点复制。
   - `h2` / `h3` hover 或 focus 时显示复制当前锚点的入口。
   - 复制成功反馈应轻量，不使用阻塞式 `alert`。

### P2：完善博客检索和列表状态

1. 搜索无结果空态。

   - 当搜索或标签过滤后没有文章时，展示明确空态。
   - 空态提供“清空搜索”和“查看全部文章”的操作。

2. 搜索建议的可访问性语义。

   - 输入框补 `role="combobox"`、`aria-expanded`、`aria-controls`。
   - 建议项使用稳定 `id`，输入框通过 `aria-activedescendant` 指向当前项。
   - 键盘方向键、Enter、Escape 行为保持现有体验，但语义要补齐。

3. 搜索命中高亮。
   - 在建议列表和文章索引中，对标题或摘要里的命中片段做轻量高亮。
   - 高亮颜色需要分别适配浅色和黑色主题。

## 简历后续

1. 移动端导出入口不要弹 `alert`。

   - 移动端目前不支持导出时，按钮应直接禁用。
   - 文案显示“请在桌面端导出 PDF”即可。

2. 补齐 `JPG` 导出。

   - `docs/codex/resume-function.md` 和 `docs/codex/resume-ui.md` 都提到 `JPG`。
   - 后续可提供 `导出 JPG 1x` 和 `导出 JPG 2x`。
   - 导出内容仍然不应包含交互控件。

3. 导出状态继续避免阻塞式反馈。
   - 成功、失败、不可导出都优先使用页面内状态。
   - 只有无法恢复的失败再考虑 `alert`。

## 首页后续

1. 增加明确导航动作。

   - 首页当前入口嵌在段落里，气质克制，但可发现性偏弱。
   - 可以在正文后增加一行轻量链接：`博客`、`简历`、`GitHub`、`主题`。
   - 不建议做大按钮或营销式 hero。

2. 保持文字站气质。
   - 首页不需要卡片化。
   - 不需要增加大面积装饰图。
   - 黑色主题下继续控制背景光斑强度，避免变成单一深蓝氛围。

## 性能与构建

1. 路由级拆包。

   - 博客和简历可以使用 `React.lazy` 分包。
   - 首屏首页不应加载简历导出和博客 Markdown 的重依赖。

2. 导出依赖延迟加载。

   - `html2canvas`、PDF 相关依赖只在点击导出时加载。
   - 避免影响普通浏览路径。

3. 字体体积收敛。
   - 当前自定义字体体积较大。
   - 移动端已经优先使用系统字体，后续可继续按页面和字重减少默认加载。

## 实施顺序建议

1. SEO/head、404 页面、全站主题按钮。
2. 博客文章目录、阅读进度、上一篇 / 下一篇。
3. 搜索空态与搜索可访问性。
4. 简历移动端导出状态和 `JPG` 导出。
5. 路由拆包、导出依赖延迟加载和字体加载优化。

每次改动保持小步提交。页面体验类改动完成后，至少运行 `pnpm build`；涉及文章内容或博客数据时，同时运行 `pnpm codex:blog-check`。
