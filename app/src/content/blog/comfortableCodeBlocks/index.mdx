# 技术博客里的代码块，不一定要做得很重

很多技术博客会把代码块做成很强的视觉模块：纯黑背景、非常亮的高亮色、很重的圆角和边框。

这不是错，但如果页面整体想走“轻、安静、阅读优先”的方向，代码块太重会打断节奏。

## 我更喜欢的代码块状态

我通常会让代码块满足这些条件：

- 和正文有清楚区分
- 但不会比正文更抢眼
- 在长文里连续出现时不显得压迫

简单说，它应该像纸上的注释区，而不是舞台中央的展示面板。

## 一个更轻的处理思路

可以只保留三件事：

- 比页面略深一点的背景
- 很轻的边框
- 比正文更稳定的等宽字

```ts
const codeStyle = {
  background: '#f2ece3',
  border: '1px solid #ddd4c7',
  borderRadius: '16px',
};
```

## 为什么这种做法更适合阅读

因为读者在技术博客里不是来“欣赏代码块”的，而是来借助代码理解上下文。

当代码块不再强行抢夺注意力时，正文和示例之间的切换会更顺，也更接近真实阅读的节奏。
