# 用标签做博客主导航

目录分类的问题在于，它总是试图给文章安排一个唯一的位置。

但真实写作里，一篇文章很可能同时属于多个上下文。

- 前端
- 工程
- 性能

如果硬把它塞进单一目录，分类本身就会变得很别扭。

## 标签更像内容的横切面

标签真正的好处，不是“更自由”，而是它允许一篇文章自然地进入多个入口。

> 同一篇文章，可以从多个认知入口被找到。

这比严格的树状分类更适合笔记型和技术型博客。

## 第一版不需要把标签做复杂

第一版只要把这几件事做好就够了：

- 稳定的标签 key
- 集中的标签展示配置
- 独立的标签详情页

后面如果要做专题页、相关文章或者更完整的标签描述，再继续往上加也不迟。
